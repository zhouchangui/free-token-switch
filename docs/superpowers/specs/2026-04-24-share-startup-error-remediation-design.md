# 分享启动过程错误处理设计

日期：2026-04-24

## 背景

V10 分享设置中，两个胶囊开关都应先显示启动过程卡片，完成后才展示结果信息：

- 好友分享：启动分享通道、生成访问令牌、生成好友导入链接。
- 卖了换钱：检查账户、启动通道、测试模型、读取定价、发布市场。

当前实现里，好友分享已有三步过程卡片，但错误只落到整体 `friend.status = "error"`；卖了换钱还没有分步骤过程卡片。下一轮实现需要做到：哪个步骤出错，就停在哪个步骤；卡片直接显示错误原因、推荐修复方案和可执行动作。

说明：用户提到的 `cloudfared` 在代码和官方工具中实际命令名为 `cloudflared`，本文统一使用 `cloudflared`。

## 总体交互

启动中卡片应保持在当前 tab 内，不弹窗。每个步骤都有四种状态：

- `pending`：未开始，灰色空心圆。
- `active`：当前执行中，loading 图标。
- `done`：已完成，绿色完成图标。
- `error`：当前步骤失败，红色错误图标，并停止后续步骤。

错误态卡片包含：

- 错误标题：一句话说明发生了什么。
- 错误详情：来自结构化错误或原始异常的可读文本。
- 修复方案：最多 2 到 3 条，优先给用户可以立即执行的动作。
- 主操作按钮：如“一键安装 cloudflared”“重新检测账户”“编辑供应商”“重试当前步骤”。
- 辅助操作：复制命令、打开帮助链接、查看日志。

启动过程卡片展示期间，当前 tab 内其它内容全部隐藏：

- 好友分享 tab 隐藏好友入口、访问令牌、复制端点和令牌、复制好友导入链接。
- 卖了换钱 tab 隐藏 ClawTip 收款账户卡、定价策略卡、市场发布结果和其它状态内容。

只有完整启动成功后，才展示该 tab 的状态内容和后续操作。错误发生后不应展示最终结果区；已经完成的步骤保持完成态，失败步骤保持错误态，后续步骤保持待执行。

## 错误展示筛选原则

启动卡片只展示最容易遇到、并且需要用户配合修复的问题。低频内部异常不展开成独立文案，统一收敛为“启动失败，请重试；如果持续失败，复制日志给开发者”。

保留到卡片里的错误应满足至少一个条件：

- 用户能立即修复：安装 `cloudflared`、绑定 ClawTip、编辑供应商、填写价格。
- 用户环境常见：网络超时、本地代理未启动、API Key 无效、额度不足。
- 不修复就无法继续：缺少收款账户、缺少公网通道、模型测试失败、价格不合法。

不单独展示的错误：

- provider id 缺失、listing 序列化失败、未知 command 异常等内部问题。
- 已完成步骤产物缺失等理论上不应发生的问题。
- 剪贴板不可用等发生在启动成功后的复制问题。

## 共享错误动作

| 动作                 | 使用场景                                               | 行为                                                         |
| -------------------- | ------------------------------------------------------ | ------------------------------------------------------------ |
| 重试当前步骤         | 网络超时、ClawTip 检测失败、模型测试超时、市场发布失败 | 从失败步骤重新开始，不重跑已成功且可复用的步骤，除非依赖失效 |
| 一键安装 cloudflared | 未检测到 `cloudflared`                                 | 调用新增 Tauri command 安装；失败时展示手动命令              |
| 复制安装命令         | 一键安装不可用或需要 sudo                              | 复制当前系统对应命令                                         |
| 重新检测             | 安装后、账户绑定后、配置修复后                         | 只重跑检测类步骤                                             |
| 编辑供应商           | API Key、Base URL、模型配置错误                        | 打开现有编辑供应商全屏面板                                   |
| 打开 ClawTip         | 收款账户未绑定、服务端密钥缺失                         | 打开 `https://clawtip.jd.com`                                |
| 启动/重启代理        | 本地 `15721` 服务不可访问                              | 执行现有代理启动/重启逻辑                                    |
| 复制日志             | 错误不可自动修复或重复失败                             | 复制最近一次步骤日志                                         |

## cloudflared 检测与安装

当前后端只在 `start_tunnel()` 中直接执行 `cloudflared tunnel --url http://localhost:15721`。如果命令不存在，会返回字符串错误：“无法启动 cloudflared...请确保已安装 cloudflared。” 下一轮应新增结构化能力：

| 能力         | 建议 command                               | 说明                                                  |
| ------------ | ------------------------------------------ | ----------------------------------------------------- |
| 检测是否安装 | `check_cloudflared`                        | 返回 `{ installed, version?, path?, installMethod? }` |
| 一键安装     | `install_cloudflared`                      | 根据系统选择安装方式，返回进度和结果                  |
| 打开安装帮助 | `open_cloudflared_install_help` 或前端外链 | 一键安装失败时兜底                                    |

安装策略建议：

- macOS：优先 Homebrew，命令 `brew install cloudflared`；如果 Homebrew 不存在，展示手动安装说明。
- Windows：优先 `winget install --id Cloudflare.cloudflared`；失败则打开 Cloudflare 下载页。
- Linux：不要默认静默 sudo；优先复制发行版命令或打开终端让用户确认执行。

## 好友分享

### 高频错误清单

| 步骤             | 最常见且需要用户配合的问题    | 卡片错误提示                                                 | 主操作               |
| ---------------- | ----------------------------- | ------------------------------------------------------------ | -------------------- |
| 启动分享通道     | 未安装 `cloudflared`          | 未检测到 cloudflared，无法创建分享通道。                     | 一键安装 cloudflared |
| 启动分享通道     | 本地代理服务没有响应          | TokensBuddy 本地代理未启动，暂时无法创建分享通道。           | 启动/重启代理        |
| 启动分享通道     | 网络超时或无法连接 Cloudflare | 创建公网通道超时，请检查网络、代理或防火墙后重试。           | 重试当前步骤         |
| 生成访问令牌     | 本地配置保存失败              | 访问令牌已生成，但无法保存到供应商配置。请检查配置文件权限。 | 重试保存             |
| 生成好友导入链接 | 本地配置保存失败              | 好友导入链接已生成，但分享配置保存失败。请检查配置文件权限。 | 重试保存             |

好友分享不需要把 token command 内部异常、链接拼接异常拆成独立错误。它们统一显示为“生成失败，请重试；如果持续失败，复制日志给开发者”。

- 好友入口。
- 访问令牌状态。
- 复制端点和令牌。
- 复制好友导入链接。

注意：以上内容仅在三步全部成功后展示；启动中或任一步骤失败时，好友分享 tab 只展示启动过程卡片。

- 市场发布状态为运行中。
- 当前 endpoint。
- 定价策略和价格。

## 结构化错误建议

不要长期依赖错误字符串匹配。建议前后端约定如下结构：

```ts
type ShareStartupStep =
  | "account"
  | "tunnel"
  | "token"
  | "link"
  | "modelTest"
  | "pricing"
  | "publish";

type ShareStartupErrorCode =
  | "cloudflared_missing"
  | "local_proxy_unavailable"
  | "network_timeout"
  | "clawtip_unbound"
  | "clawtip_secret_missing"
  | "clawtip_wallet_not_ready"
  | "provider_config_invalid"
  | "provider_quota_or_rate_limited"
  | "pricing_invalid"
  | "pricing_unavailable"
  | "market_publish_failed"
  | "persist_failed"
  | "unknown";

interface ShareStartupError {
  step: ShareStartupStep;
  code: ShareStartupErrorCode;
  title: string;
  detail: string;
  retryable: boolean;
  primaryAction?:
    | "retry"
    | "install_cloudflared"
    | "open_clawtip"
    | "edit_provider"
    | "start_proxy";
}
```

前端可以先做字符串分类兜底，但新增 Tauri command 时应返回结构化错误。

## 需要补齐的实现能力

1. 为好友分享和卖了换钱分别维护 `startupStep`、`startupError`、`completedSteps`。
2. 卖了换钱新增五步启动过程卡片。
3. 好友分享错误时不再只显示普通错误块，而是在过程卡片中停留在失败步骤。
4. 新增 `check_cloudflared` 和 `install_cloudflared`，至少先实现检测和手动命令复制；一键安装可按平台渐进实现。
5. 新增或复用模型测试能力，确保“测试模型”步骤不是假步骤。
6. 新增 ClawTip 账户检测能力；在真实 API 未接入前，未绑定卡片只能引导开通，不能声称检测已通过。
7. 定价读取需要明确单位换算：UI 是 USTC / 百万 TOKEN，后端当前 `start_selling_tokens` 使用 `price` 字段，现有注释为 sats per 1k tokens。实现前必须统一单位。
8. 发布市场错误需要保留 endpoint、token、定价草稿，方便重试。

## 待确认点

1. `cloudflared` 一键安装是否允许在 macOS/Windows 直接执行包管理器命令，Linux 是否只复制命令并打开终端确认。
2. ClawTip 账户状态是否已有可调用 API；如果没有，第一版是否只做“未绑定引导 + 手动重新检测占位”。
3. “测试模型”应调用现有供应商测试能力，还是新增一个更轻量的 market 发布前测试 command。
4. 自定义定价最终单位以 UI 的 USTC / 百万 TOKEN 为准，还是沿用后端当前的 sats / 1K tokens。
